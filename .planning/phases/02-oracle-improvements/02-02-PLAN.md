<objective>
Add path-based fingerprinting to the compiler oracle validation.

Purpose: The current `action_bag` comparison characterizes graphs by counting node neighbourhood multisets, losing edge order and path information. Two structurally different graphs (e.g., deterministic vs non-deterministic compilation of the same if-statement) can produce identical action bags. Path fingerprinting walks every path from the start node and compares ordered action label sequences, catching structural differences that action_bag misses.

Output: A `path_fingerprints` function in `lib.rs` and an augmented `validate()` that runs both action_bag and path fingerprint comparison.
</objective>

**Add a `path_fingerprints` function** to `crates/envs/ce-compiler/src/lib.rs`, placed after the `action_bag` function:

```rust
const MAX_PATHS: usize = 512;

fn path_fingerprints(g: &dot::ParsedGraph) -> Option<BTreeSet<Vec<String>>> {
    let start = g.node_mapping.get("qStart")?;
    let mut all_paths = BTreeSet::new();
    // Stack: (current_node, path_so_far, visited_nodes)
    let mut stack: Vec<(petgraph::graph::NodeIndex, Vec<String>, BTreeSet<petgraph::graph::NodeIndex>)> = vec![
        (*start, vec![], BTreeSet::new()),
    ];
    while let Some((node, path, visited)) = stack.pop() {
        if all_paths.len() >= MAX_PATHS {
            return None; // Exceeded limit — caller should fall back to action_bag only
        }
        let outgoing: Vec<_> = g.graph.edges(node).collect();
        if outgoing.is_empty() {
            all_paths.insert(path);
        } else {
            for edge in outgoing {
                let target = edge.target();
                if !visited.contains(&target) {
                    let mut new_path = path.clone();
                    new_path.push(edge.weight().to_string());
                    let mut new_visited = visited.clone();
                    new_visited.insert(node);
                    stack.push((target, new_path, new_visited));
                }
            }
        }
    }
    Some(all_paths)
}
```

Key design decisions:
- Returns `Option<BTreeSet<Vec<String>>>` — `None` when qStart is missing or MAX_PATHS exceeded.
- Uses `BTreeSet<Vec<String>>` for canonical, order-independent comparison across paths while preserving edge-label order within each path.
- Tracks visited nodes per-path (not globally) to handle cycles correctly — a `do` loop's back-edge creates a cycle boundary per-path.
- Uses `edge.weight().to_string()` via `Action`'s `Display` impl for canonical edge labels. Does NOT include node names (which differ between reference and student).
- `MAX_PATHS = 512` constant prevents combinatorial explosion.

**Add necessary imports** at the top of `lib.rs` if not already present:

```rust
if action_bag(&o_g, &sample_mems) != action_bag(&t_g, &sample_mems) {

// (e.g., deterministic vs non-deterministic compilation)
match (path_fingerprints(&o_g), path_fingerprints(&t_g)) {
    (Some(o_paths), Some(t_paths)) if o_paths != t_paths => {
        Ok(ValidationResult::Mismatch {
            reason: "path fingerprints differ — graph structure mismatch".to_string(),
        })
    }
    // If either side returns None (no qStart or path limit exceeded),
    // we already passed action_bag — accept as correct
    _ => Ok(ValidationResult::Correct),
}
```

This is an augmentation: action_bag runs first (fast, catches most errors), then path_fingerprints runs as a second pass (catches determinism errors). If path enumeration exceeds MAX_PATHS on either graph, we fall back to the action_bag result (which already passed).

If `qStart` is not found in the student graph's `node_mapping`, `path_fingerprints` returns `None`. The match arm catches this and falls through to `Correct` (since action_bag already passed). The reference graph always has `qStart` (generated by `ProgramGraph::dot()`).

```rust
#[test]
fn path_fingerprints_differ_for_det_vs_nondet() {
    use gcl::ast::Commands;
    use gcl::pg::{Determinism, ProgramGraph};

    // Program with overlapping guards — the key failure case
    let commands: Commands = "if a > 0 -> x := 1 [] a < 10 -> x := 2 fi"
        .parse()
        .unwrap();

    let det_dot = ProgramGraph::new(Determinism::Deterministic, &commands).dot();
    let nondet_dot = ProgramGraph::new(Determinism::NonDeterministic, &commands).dot();

    let det_g = dot::dot_to_petgraph(&det_dot).unwrap();
    let nondet_g = dot::dot_to_petgraph(&nondet_dot).unwrap();

    let det_paths = path_fingerprints(&det_g);
    let nondet_paths = path_fingerprints(&nondet_g);

    // Both should produce valid path sets (not None)
    assert!(det_paths.is_some(), "det graph should have qStart and paths");
    assert!(nondet_paths.is_some(), "nondet graph should have qStart and paths");

    // The path fingerprints MUST differ — deterministic compilation produces
    // "b1 && !false" / "b2 && !b1" guards while non-deterministic produces
    // plain "b1" / "b2" guards
    assert_ne!(
        det_paths.unwrap(),
        nondet_paths.unwrap(),
        "Path fingerprints should differ between det and nondet compilation of overlapping guards"
    );
}
```
