**TGT-08: gen_array_assignment** — Guarantee `A[expr] := expr` (array on LHS).

This constructs `Target::Array(...)` DIRECTLY as the assignment LHS, bypassing `cx.no_arrays`. The research doc's Pattern 5 explains why: targeted generators force constructs by direct AST construction, not by toggling context flags.

```rust
/// TGT-08: Array assignment — `A[expr] := expr`.
///
/// Constructs `Target::Array` directly as the LHS, ignoring `cx.no_arrays`.
/// The array name comes from `cx.array_names` (default: "A", "B", "C").
fn gen_array_assignment<R: Rng>(cx: &mut CompilerContext, rng: &mut R) -> Commands {
    let arr_name = cx
        .array_names
        .choose(rng)
        .cloned()
        .unwrap_or_else(|| "A".into());
    let idx = gen_aexpr(cx, rng);
    let rhs = gen_aexpr(cx, rng);
    let guaranteed = Command::Assignment(
        Target::Array(Array(arr_name), Box::new(idx)),
        rhs,
    );
    Commands(vec![guaranteed])
}
```

Note: Uses `.choose(rng)` (from `IndexedRandom`, already imported) to pick a random array name from the pool for variety, not always "A".

**TGT-10: gen_non_deterministic_overlapping** — Guarantee overlapping guard conditions.

IMPORTANT (Pitfall 6 from research): Determinism is NOT controlled here. The generator only produces AST with overlapping boolean conditions. Whether those become overlapping edges depends on `input.determinism` chosen at the caller level in `ce-compiler/src/lib.rs` (line 104-106). The generator's job is to produce AST where two guards' conditions can BOTH be true simultaneously.

```rust
/// TGT-10: Overlapping guards for non-deterministic mode.
///
/// Produces `if (var > 0) -> cmd1 [] (var < 10) -> cmd2 fi`.
/// Both guards are true when `0 < var < 10`, creating non-deterministic
/// choice. Whether the program graph uses deterministic or non-deterministic
/// edges is decided by `input.determinism` at the caller — NOT here.
fn gen_non_deterministic_overlapping<R: Rng>(cx: &mut CompilerContext, rng: &mut R) -> Commands {
    let var_name = cx
        .names
        .choose(rng)
        .cloned()
        .unwrap_or_else(|| "a".into());
    let var_ref = AExpr::Reference(Target::Variable(Variable(var_name)));

    // Guard 1: var > 0
    let b1 = BExpr::Rel(var_ref.clone(), RelOp::Gt, AExpr::Number(0));
    // Guard 2: var < 10
    let b2 = BExpr::Rel(var_ref, RelOp::Lt, AExpr::Number(10));

    let g1 = Guard(
        b1,
        Commands(vec![Command::Assignment(gen_target(cx, rng), gen_aexpr(cx, rng))]),
    );
    let g2 = Guard(
        b2,
        Commands(vec![Command::Assignment(gen_target(cx, rng), gen_aexpr(cx, rng))]),
    );
    Commands(vec![Command::If(vec![g1, g2])])
}
```

Note: `var_ref.clone()` works because `AExpr` derives `Clone` (see ast.rs line 71).

**TGT-11: gen_variable_reuse** — Guarantee same variable assigned in multiple guards.

```rust
/// TGT-11: Same variable assigned in multiple guards.
///
/// Produces `if b1 -> var := e1 [] b2 -> var := e2 fi` where both guard
/// bodies assign to the same `Variable`. This stresses the compiler's
/// handling of variable liveness across non-deterministic branches.
fn gen_variable_reuse<R: Rng>(cx: &mut CompilerContext, rng: &mut R) -> Commands {
    let var_name = cx
        .names
        .choose(rng)
        .cloned()
        .unwrap_or_else(|| "a".into());
    // Closure that builds the same target each time
    let make_target = || Target::Variable(Variable(var_name.clone()));

    let g1 = Guard(
        gen_bexpr(cx, rng),
        Commands(vec![Command::Assignment(make_target(), gen_aexpr(cx, rng))]),
    );
    let g2 = Guard(
        gen_bexpr(cx, rng),
        Commands(vec![Command::Assignment(make_target(), gen_aexpr(cx, rng))]),
    );
    Commands(vec![Command::If(vec![g1, g2])])
}
```

**TGT-12: gen_variable_as_index** — Guarantee variable used as both array index and scalar.

```rust
/// TGT-12: Variable used as both array index and scalar.
///
/// Produces two commands:
/// 1. `A[a] := expr` — variable `a` used as array index
/// 2. `a := expr`    — same variable `a` used as scalar target
///
/// This exercises the compiler's handling of a variable that appears in
/// both scalar and array-index positions within the same program.
fn gen_variable_as_index<R: Rng>(cx: &mut CompilerContext, rng: &mut R) -> Commands {
    let var_name = cx
        .names
        .choose(rng)
        .cloned()
        .unwrap_or_else(|| "a".into());
    let arr_name = cx
        .array_names
        .choose(rng)
        .cloned()
        .unwrap_or_else(|| "A".into());

    // Command 1: A[a] := expr  (variable as array index)
    let idx = AExpr::Reference(Target::Variable(Variable(var_name.clone())));
    let arr_assign = Command::Assignment(
        Target::Array(Array(arr_name), Box::new(idx)),
        gen_aexpr(cx, rng),
    );

    // Command 2: a := expr  (same variable as scalar target)
    let scalar_assign = Command::Assignment(
        Target::Variable(Variable(var_name)),
        gen_aexpr(cx, rng),
    );

    Commands(vec![arr_assign, scalar_assign])
}
```
